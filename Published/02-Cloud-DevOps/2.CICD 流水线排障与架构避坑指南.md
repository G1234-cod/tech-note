**2. VitePress SSR 编译期的正则误伤陷阱**

- **底层原理剖析**：VitePress 配置文件中的 `srcExclude: ['/*_*.md']` 原本是为了规避某些插件生成的临时文件。但这种粗暴的通配符，会在 AST 抓取阶段将所有合法且带有下划线的 Markdown 文件从路由树中静默剔除。

```javascript
srcExclude: [
  '**/模板.md', 
  '**/阅读笔记_参考模板.md', 
  '**/README.md', 
  '**/LeetCode-*.md'
]

```
