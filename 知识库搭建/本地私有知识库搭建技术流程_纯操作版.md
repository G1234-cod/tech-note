# 本地私有知识库搭建技术流程（纯操作版）

> 本文是一份无废话的实操指南，涵盖从零开始部署 Ollama + AnythingLLM 本地知识库的全部命令和步骤。适合已经了解基本概念、需要照着执行的读者。

## 环境概况

- 宿主机：Windows 11 + WSL2（Ubuntu 24.04）
- 容器运行时：Docker Desktop（已集成 WSL2 后端）
- 模型服务：Ollama（宿主机直接安装）
- 知识库应用：AnythingLLM（通过 Docker 部署）

## 第一步：WSL2 环境网络修复（如遇 DNS 问题执行）

```bash
# 修改 DNS
sudo nano /etc/resolv.conf
# 清空内容，写入：
nameserver 223.5.5.5
nameserver 8.8.8.8

# 禁止自动生成
sudo nano /etc/wsl.conf
# 添加：
[network]
generateResolvConf = false

# 退出 WSL，在 Windows 终端执行：
wsl --shutdown
```

重新打开 WSL 终端后网络恢复正常。

## 第二步：安装 Ollama 并拉取模型

```bash
# 安装 zstd 依赖
sudo apt update && sudo apt install zstd -y

# 安装 Ollama（如果网络不稳可加 --http1.1）
curl --http1.1 --retry 5 -fsSL https://ollama.com/install.sh | sh

# 配置监听所有接口（方便容器访问）
sudo systemctl edit ollama.service
# 输入以下内容并保存：
[Service]
Environment="OLLAMA_HOST=0.0.0.0:11434"

# 重启服务
sudo systemctl daemon-reload
sudo systemctl restart ollama

# 拉取所需模型
ollama pull qwen2.5:7b
ollama pull nomic-embed-text
```

## 第三步：获取 AnythingLLM 镜像（国内加速）

```bash
# 方式一：从镜像站直接拉取并打标签（推荐）
docker pull docker.1ms.run/mintplexlabs/anythingllm:latest
docker tag docker.1ms.run/mintplexlabs/anythingllm:latest mintplexlabs/anythingllm:latest

# 方式二：配置 Docker Desktop 加速器后再拉取
# 在 Docker Desktop 的 Settings -> Docker Engine 中添加：
{
  "registry-mirrors": [
    "https://docker.xuanyuan.me",
    "https://docker.1ms.run",
    "https://docker.m.daocloud.io"
  ]
}
# 点击 Apply & Restart，然后执行：
docker pull mintplexlabs/anythingllm:latest

# 清理可能残留的无效数据
docker system prune -a -f
```

## 第四步：编写配置文件并启动 AnythingLLM

```bash
# 创建项目目录
mkdir -p ~/anythingllm && cd ~/anythingllm

# 创建 .env 文件（开启认证，生成默认管理员）
nano .env
# 内容如下（密码请自行修改）：
ENABLE_AUTH=true
DEFAULT_USER_EMAIL=admin@local.ai
DEFAULT_USER_PASSWORD=MySecurePass!2024
```

编辑 `docker-compose.yml`：

```bash
nano docker-compose.yml
```

粘贴以下内容：

```yaml
services:
  anythingllm:
    image: mintplexlabs/anythingllm:latest
    container_name: anythingllm
    ports:
      - "3001:3001"
    volumes:
      - ~/knowledge_base:/app/server/storage
    environment:
      - STORAGE_DIR=/app/server/storage
    restart: unless-stopped
    extra_hosts:
      - "host.docker.internal:host-gateway"
```

启动容器：

```bash
docker compose up -d
```

检查状态：

```bash
docker ps
docker logs anythingllm   # 应看到 “Server is running on port 3001”
```

## 第五步：浏览器初始化配置

1. 在宿主机浏览器访问 `http://localhost:3001`
2. 如果开启了认证，用 `.env` 中设置的邮箱和密码登录
3. 在 LLM 偏好设置中选择 **Ollama**
   - **Ollama Base URL**：`http://host.docker.internal:11434`
   - **Chat Model**：`qwen2.5:7b`
   - **Embedding Model**：`nomic-embed-text`
4. 创建两个工作区（例如 “私密知识库” 和 “共享知识库”），它们的数据将自动存储在宿主机 `~/knowledge_base` 下的对应子目录中

## 第六步：多用户权限隔离（可选）

- 进入管理后台，开启多用户模式
- 为每个用户分配角色（Manager 或普通用户）
- 将用户关联到指定工作区，实现文档内容完全隔离
- 为“共享知识库”生成 API Key，供外部软件调用；私密工作区的 Key 只分发给管理员

## 第七步：迁移到服务器与升级模型

**迁移要点：**

- 备份 `~/knowledge_base` 整个目录
- 修改 `docker-compose.yml` 中的卷挂载路径为服务器绝对路径（如 `/data/anythingllm/storage`）
- 如果 Ollama 分离部署，删掉 `extra_hosts`，并在界面中填写真实 IP
- 如需 GPU 加速，添加 `deploy.resources` 配置块

**升级模型：**

```bash
# 拉取新模型（例如 14B）
ollama pull qwen2.5:14b
# 在 AnythingLLM 界面中直接切换模型，无需重启容器
```
